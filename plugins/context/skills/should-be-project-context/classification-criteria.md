# 分割基準

コンテキストファイルをどこに配置すべきかの判断基準。

## コロケーションの原則

ディレクトリ固有のルールやガイドラインは、`.claude/rules/` に配置するよりも **そのディレクトリに AGENTS.md を作成**する方が好ましい。

理由:

- 実装とドキュメントが近接することで、変更時に更新漏れが起きにくい
- そのディレクトリで作業する際に自動的に読み込まれる
- ディレクトリの責務が明確になる

`.claude/rules/` が適切な場合:

- プロジェクト全体に適用されるルール
- 複数ディレクトリにまたがる制約
- パス条件（`paths:` frontmatter）で柔軟に制御したい場合

## AGENTS.md 階層設計（モノレポの場合）

### ルート AGENTS.md に書くべき内容

- プロジェクト概要（2-3文）
- ディレクトリ/パッケージ構成（1行説明のみ）
- 基本コマンド一覧（日常的に使うもの）
- 重要な指示・制約
- Skills 一覧
- コマンド最終更新日

### ルート AGENTS.md の禁止事項

- 各パッケージ/プロジェクトの詳細な使用方法
- 環境変数の詳細
- 具体的なコード例
- API仕様
- ルールへの参照（適切に設定できていれば自動ロードされるので不要）

### 各パッケージ/プロジェクトの AGENTS.md に書くべき内容

- パッケージ/プロジェクト固有の詳細情報
- 開発コマンド（全て）
- 環境変数の一覧と説明
- 具体的なコマンド例
- ディレクトリ構造の詳細
- API仕様・エクスポート一覧
- 重要事項・注意点
- コマンド最終更新日

### ルール化すべき内容

- プロジェクト固有の用語集 → `project-glossary.md`
- 参照リンク → `references.md`
- 複数のプロジェクト・パッケージに影響するが、記述が長くなるもの
  - 例: コーディング規約（サンプルコードが必要になるため）
  - 例: アクセシビリティ規約（WCAG準拠、コントラスト比など詳細なガイドラインが必要になるため）
