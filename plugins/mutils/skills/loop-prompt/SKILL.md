# loop-prompt

`/loop` で繰り返し実行する際に、前イテレーションの知識による手順省略・簡略化を防ぎ、毎回完全な処理を強制するプロンプトを生成する。

## フロー

```text
1. $ARGUMENTS からタスク概要を取得
2. AskUserQuestion でヒアリング（4問）
3. ヒアリング結果をテンプレートに埋め込み
4. コピー可能なコードブロックで出力
```

## Phase 1: ヒアリング

`$ARGUMENTS` の内容を踏まえ、AskUserQuestion で以下の4問を同時に質問する。

質問1 — **完了条件**: このloopはいつ終了すべきですか？（例: 「全PRがマージされたら」「エラーが0件になったら」「3回連続成功したら」）

- 選択肢は $ARGUMENTS の内容から推測して2-3個生成し、最も妥当なものを先頭に置く

質問2 — **毎回再確認すべき対象**: 各イテレーションで毎回ゼロから確認すべきファイル・コンテキスト・状態は何ですか？

- 選択肢は $ARGUMENTS の内容から推測して2-3個生成する（例: 「git status + CI結果」「特定のファイル群」「PRのレビューコメント」）

質問3 — **毎回再実行すべきコマンド**: 各イテレーションで毎回実行すべきコマンドは何ですか？

- 選択肢は $ARGUMENTS の内容から推測して2-3個生成する（例: 「lint + test」「build + deploy」「gh pr checks」）

質問4 — **省略されやすいステップ**: 過去の経験で省略されがちなステップがあれば教えてください

- 選択肢: 「特になし」「具体的に指定する（Otherで入力）」

## Phase 2: プロンプト生成

ヒアリング結果を以下のテンプレートに埋め込み、**コードブロック内にプロンプト全文を出力**する。ユーザーがそのままコピーして `/loop` に貼り付けられる形式にする。

MUST: テンプレートの各セクションを省略しない
MUST: ヒアリングで得た具体的な内容をそのまま埋め込む（抽象化しない）
MUST: 出力はコードブロック1つのみ（説明文は前後に最小限）
