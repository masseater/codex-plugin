ステアリングドキュメントに基づいてコードやドキュメントの問題点を指摘してください。

**修正作業は行わず、指摘のみを行います。**

## 役割

- **オーケストレーター**: 各チェック観点ごとにサブエージェントを並列起動
- **指摘専門**: プロジェクト方針との乖離を発見して報告するのみ（修正は行わない）
- **レポート統合**: 各サブエージェントの結果を統合してレポート生成

## 実行手順

### 1. 対象ファイルの特定

```bash
# ステアリングドキュメント
Read: .claude/skills/steering/SKILL.md

# レビュー対象（ユーザー指示に基づく）
Glob: specs/[taskname]/**/*.md
# または対象コードファイル
```

### 2. サブエージェントによる並列チェック

以下の4つのチェックを **Task ツールで並列実行**:

| チェック   | チェック項目                                               |
| ---------- | ---------------------------------------------------------- |
| Product    | ビジネス目標・ターゲットユーザー・ビジョンとの整合性       |
| Tech Stack | 技術スタック・アーキテクチャ・コーディング標準・テスト戦略 |
| Structure  | ディレクトリ構造・命名規則・モジュール境界・ファイル配置   |
| Principles | TDD・SOLID・YAGNI などの開発原則                           |

各サブエージェントへのプロンプト例:

```
.claude/skills/steering/SKILL.md と specs/[taskname]/ を読み込み、
Product方針との整合性をチェックしてください。

チェック項目:
- ビジネス目標に沿った実装か
- ターゲットユーザーの要求を満たしているか
- プロダクトのビジョンと矛盾していないか

結果フォーマット:
【チェック項目】Product方針との整合性
【乖離数】X件
【詳細】
  - 乖離1: [ファイル:行番号] - 説明
```

### 3. 結果の統合とレポート生成

```markdown
## ステアリングドキュメントレビュー結果

### ✅ 準拠している点

- [準拠している点をリストアップ]

### ⚠️ 指摘事項

#### 1. [問題のカテゴリ]

**ファイル**: `path/to/file.ts:123`
**違反内容**: [ステアリングのどのセクション・項目に違反しているか]
**現状**: [現在の実装]
**期待**: [ステアリングドキュメントで定義されている期待値]

### 📊 サマリー

- 準拠項目: X件
- 指摘事項: Y件
```
