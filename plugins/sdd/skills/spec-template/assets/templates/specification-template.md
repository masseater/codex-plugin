# specification.md テンプレート

機能要件と非機能要件を定義する。

```markdown
# [タスク名] 要件定義

## 機能要件

### FR-1: [機能名]

- **説明**: [機能の説明]
- **入力**: [入力データ]
- **出力**: [出力データ]
- **前提条件**: [実行に必要な条件]
- **事後条件**: [実行後の状態]

### FR-2: [機能名]

[...]

## 非機能要件

### NFR-1: セキュリティ（必須）

- **要件**: [セキュリティ要件]
- **対策**: [具体的な対策]

### NFR-2: パフォーマンス

- **要件**: [パフォーマンス要件]
- **指標**: [測定指標]
- **目標値**: [目標値]

### NFR-3: 可用性

- **要件**: [可用性要件]
- **目標**: [目標値]

### NFR-4: 保守性

- **要件**: [保守性要件]
- **方針**: [具体的な方針]

## ユースケース

### UC-1: [ユースケース名]

- **アクター**: [アクター]
- **前提条件**: [前提]
- **基本フロー**:
  1. [ステップ1]
  2. [ステップ2]
  3. [ステップ3]
- **代替フロー**: [代替フロー]
- **例外フロー**: [例外フロー]

## 制約事項

- [制約1]
- [制約2]

## 用語定義

| 用語    | 定義   |
| ------- | ------ |
| [用語1] | [定義] |
| [用語2] | [定義] |
```

## 注意事項

- **セキュリティは常に含める**（必須NFR）
- 他のNFRはプロジェクトに応じて選択
- ステアリングドキュメント（product.md）と整合性を保つ
- 不明な要件は「**不明**」とマーク
