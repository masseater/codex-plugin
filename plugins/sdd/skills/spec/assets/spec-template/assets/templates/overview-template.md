# overview.md テンプレート

タスク概要、調査項目、Phase構成を記載する。

## init-task後（実装概要・調査項目も含む）

```markdown
# [タスク名]

## 目的

[何を達成したいのか 1-3行]

## スコープ

### 含まれるもの

- [項目1]
- [項目2]

### 含まれないもの

- [除外項目1]
- [除外項目2]

## 想定される成果物

- [成果物1]
- [成果物2]

## 実装概要

### 実装する機能

1. [機能名1]
   - 概要: [1-2行で説明]
   - 入力: [概要レベル]
   - 出力: [概要レベル]
   - 優先度: High/Medium/Low

### 技術スタック（候補）

- フロントエンド: [候補]
- バックエンド: [候補]
- データベース: [候補]

## 調査項目

| #   | 調査項目    | 状態      | 結果ファイル |
| --- | ----------- | --------- | ------------ |
| 1   | [調査項目1] | 🔴 未着手 | -            |
| 2   | [調査項目2] | 🔴 未着手 | -            |

---

**次のステップ**: `/sdd:research conduct [taskname]`（調査項目がある場合）または `/sdd:spec requirements [taskname]`
```

## plan-phases後（Phase構成追加）

```markdown
# [タスク名]

[...上記セクション...]

## Phase構成

| Phase | 名称      | 目標   | 依存    | 成果物   |
| ----- | --------- | ------ | ------- | -------- |
| 1     | [Phase名] | [目標] | -       | [成果物] |
| 2     | [Phase名] | [目標] | Phase 1 | [成果物] |
| 3     | [Phase名] | [目標] | Phase 2 | [成果物] |

### Phase詳細

#### Phase 1: [名称]

- **目標**: [目標]
- **成果物**: [成果物]
- **状態**: 未着手

#### Phase 2: [名称]

[...]

---

**次のステップ**: `/sdd:phase breakdown [taskname] 1`
```

## 注意事項

- 「将来的に必要になる」という推測に基づく記述は禁止
- 不明な点は「**不明**」と明記
- 調査項目の状態は正確に更新する
