```markdown
# Phase {N}: [Phase名]

## 概要

- **目標**: [このPhaseで達成すること]
- **依存**: [依存するPhase]
- **成果物**: [このPhaseの成果物]

| #     | タスク     | TDDステップ | 状態      |
| ----- | ---------- | ----------- | --------- |
| {N}.1 | [タスク名] | Red         | ⬜ 未着手 |
| {N}.2 | [タスク名] | Green       | ⬜ 未着手 |
| {N}.3 | [タスク名] | Refactor    | ⬜ 未着手 |
| {N}.4 | [タスク名] | Red         | ⬜ 未着手 |

## タスク詳細

### {N}.1: [タスク名]

- **説明**: [タスクの詳細説明]
- **TDDステップ**: Red（失敗するテストを書く）
- **入力**: [入力となる成果物やデータ]
- **出力**: [このタスクの成果物]
- **受け入れ基準**:
  - [ ] [基準1]
  - [ ] [基準2]

### {N}.2: [タスク名]

- **説明**: [タスクの詳細説明]
- **TDDステップ**: Green（テストを通す最小実装）
- **入力**: [入力]
- **出力**: [出力]
- **受け入れ基準**:
  - [ ] [基準1]
  - [ ] [基準2]

### {N}.3: [タスク名]

- **説明**: [タスクの詳細説明]
- **TDDステップ**: Refactor（リファクタリング）
- **入力**: [入力]
- **出力**: [出力]
- **受け入れ基準**:
  - [ ] [基準1]
  - [ ] [基準2]

## 依存関係
```

{N}.1 (テスト作成)
↓
{N}.2 (実装)
↓
{N}.3 (リファクタリング)
↓
{N}.4 (次の機能のテスト)
...


## タスク状態の表記

| 状態     | 表記        |
| -------- | ----------- |
| 未着手   | ⬜ 未着手   |
| 進行中   | 🔄 進行中   |
| 完了     | ✅ 完了     |
| ブロック | 🚫 ブロック |

## TDDステップの説明

| ステップ | 説明                         |
| -------- | ---------------------------- |
| Red      | 失敗するテストを書く         |
| Green    | テストを通す最小限の実装     |
| Refactor | コードを改善（テストは維持） |
