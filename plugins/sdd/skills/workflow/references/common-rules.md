# 共通ルール

すべてのSDDコマンドで適用される共通ルール。

## 記載ルール

- **⚠️ 詳細 ≠ 品質**: 必要な部分だけを必要なだけ書く
- **⚠️ 推測禁止**: 「将来的に必要」「今後〜が必要」という推測に基づく記述は禁止
- **⚠️ 不明は明記**: 勝手に決めず「**不明**」と明記し、複数の案（案A、案B、案C）を提示
- **⚠️ 工数禁止**: 開発期間、工数、見積もり時間は一切記載しない

## YAGNI原則

- 実際に必要になるまで機能を追加しない
- 過度な抽象化を避ける
- インターフェースや抽象クラスは、それなしでは実現できない時のみ使用
- 型定義だけを先に作ることは禁止（実装と共に作成）

## ステアリングドキュメント参照

ステアリングドキュメント（`.claude/skills/steering/SKILL.md`）は Claude の Progressive Disclosure により文脈に応じて自動参照される。
